# Video Scriptwriter — Win the First 3 Seconds, Then Keep Them

Video is a **retention** game: you have seconds to hook a viewer and must give them a reason to stay at every beat. A script isn't just narration — it's narration *plus* what's on screen, paced to never let attention drift. Write for the ear and the eye together, kill every dull moment, and match the structure to the platform (a 45-second Short and a 12-minute YouTube essay are different crafts).

## What this skill produces

**Chat-first.** Default: the script with narration **and visual/B-roll cues** (two-column or inline), a hook, beat structure, and CTA — formatted for the target platform. Apply the creator's voice (→ `brand-voice`). For YouTube metadata (title/description/tags) pair with `youtube-content`.

## When to use

- "Write a video script / YouTube script / Shorts/TikTok/Reel script for \<topic\>."
- "Write a hook for this video." / "How do I open?"
- "Storyboard / outline this video."
- "Make this script more engaging / improve retention."
- "Script an explainer / tutorial / ad."

## The hook (the most important 3–10 seconds)

Most viewers leave in the first few seconds — the hook decides the video's fate. Lead with the payoff, not a slow intro ("Hey guys, welcome back, don't forget to subscribe" loses people). Strong hook patterns:
- **Bold claim / result** ("I doubled my output with one change").
- **Curiosity gap / open loop** ("The mistake in this code took down a $1B company").
- **Question the viewer feels** ("Why does your code work locally but break in prod?").
- **Stakes / preview** ("By the end you'll be able to X").
- **Pattern interrupt / visual** (start mid-action, surprising image).
**Never bury the lede** behind a logo animation or long greeting. Deliver value or tension immediately, and on a long video, tease what's coming.

## Structure by format

- **YouTube long-form (5–15+ min):** Hook → (re-)promise/context → body in clear segments, each with its own mini-hook → escalating value → payoff → CTA. Use **open loops** ("more on that in a sec") to pull viewers across segments. Pacing: cut dead air, vary energy.
- **Short-form (15–60s, Shorts/Reels/TikTok):** Hook in frame 1 (no intro), one idea, fast pace, payoff before they swipe, often a loop or a "follow for part 2". Every second counts; no throat-clearing.
- **Explainer/tutorial:** Hook (the problem/outcome) → overview → ordered steps (one concept each, show don't just tell) → recap → CTA. Clarity > flash.
- **Ad:** Hook (problem/pattern interrupt) → agitate → product as solution → proof → CTA. Short and benefit-led.

## Narration + visuals together

- **Write the visual cue for every line** — B-roll, on-screen text, demo, cut, graphic. Video that's just a talking head reading is low-retention; the visuals carry half the attention.
- **Show, don't tell:** demonstrate the thing rather than describing it; on-screen text reinforces key points.
- **Pace with edits:** plan cuts/zooms/visual changes every few seconds to reset attention (the "every 3–5s something changes" rule).
- **Conversational narration:** write for the spoken voice (contractions, short sentences, direct "you"), not formal prose. Read it aloud — if it's a mouthful, rewrite.

## Retention tactics

- **Open loops** — promise something later, pay it off ("I'll show you the fix at the end — but first…").
- **Mini-hooks per segment** so each section re-earns the stay.
- **Cut everything boring** — if a line doesn't inform, entertain, or advance, delete it. Tight beats long.
- **Vary rhythm** — alternate pace/energy; monotony loses people.
- **Payoff** — deliver the promised value; don't bait without paying off (kills trust and the channel).

## CTA

One clear ask placed where it fits (subscribe, comment a question, link in description, next video). On long videos, a soft mid-roll + a clear end CTA; on shorts, often "follow for more". Don't front-load "subscribe" before delivering value.

## Chat output format

```
**Script — "Why your prod breaks" (YouTube, ~6 min)**

[0:00 HOOK]
NARRATION: Your code works perfectly on your machine. Then it hits prod and
explodes. Here's the one reason — and how to never get burned again.
VISUAL: split screen "works locally ✅ / prod 💥", fast zoom.

[0:12 PROMISE]
NARRATION: By the end you'll have a 3-step checklist...
VISUAL: checklist animates in.

[0:30 SEGMENT 1 — Environment drift]  (mini-hook: "this one's sneaky")
NARRATION: ...
VISUAL: terminal demo, on-screen text "ENV ≠ ENV".
...
[END CTA]
NARRATION: If this saved you a 2am page, subscribe — next video: the deploy checklist.
VISUAL: end card + subscribe.
```

## Workflow

1. **Pin format + length + platform** and the single core message (and the creator's voice → `brand-voice`).
2. **Write the hook first** (3–5 options) — it's the highest-leverage line.
3. **Outline beats** with a mini-hook per segment and open loops connecting them.
4. **Draft narration + a visual cue for every line**; write conversationally (read aloud).
5. **Tighten for retention** — cut dead lines, vary pace, ensure the payoff lands; add the CTA.
6. **Deliver** the formatted script; pair with `youtube-content` (title/desc/tags), `podcast-production` (clips), `newsletter-editor` (cross-promo).

## Key pitfalls

- **Slow open / buried lede.** "Welcome back, smash subscribe" before any value loses viewers in seconds — hook immediately.
- **Narration with no visuals.** A script that's only spoken words ignores half the medium — cue B-roll/text/demos per line.
- **One pace, no cuts.** Monotony kills retention — vary energy and plan a visual change every few seconds.
- **Wrong structure for the platform.** A long intro on a Short, or a Short's frantic pace on an explainer — match the format.
- **Bait without payoff.** Clickbait you don't deliver destroys trust and the algorithm punishes it.
- **Formal/written prose as narration.** Sounds robotic spoken — write conversationally and read aloud.
- **No clear CTA, or it's front-loaded.** One ask, placed after value.
- **Trying to say everything.** One core message per video; cut the rest.

## Quick reference

- Retention game: win the first 3–10s (hook = payoff/curiosity/stakes, no slow intro), then re-earn every beat.
- Structure by format: long-form (hook→promise→segmented body w/ mini-hooks→payoff→CTA), short-form (hook frame 1, one idea, fast, loop), explainer (problem→steps→recap), ad (hook→agitate→solution→proof→CTA).
- Narration + a visual cue per line; show don't tell; visual change every few seconds; write for the spoken voice (read aloud).
- Open loops, mini-hooks, cut everything boring, vary pace, deliver the payoff. One core message, one clear CTA.
- Voice → brand-voice; title/desc/tags → youtube-content; clips → podcast-production.
